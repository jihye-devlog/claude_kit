## 세션 시작
새 세션이 시작되면 .claude/memory/MEMORY.md 를 읽고 링크된 메모리 파일을 모두 읽어 이전 진행 상황을 파악한다.
메모리를 업데이트할 때는 .claude/memory/ 디렉토리의 파일을 수정한다.
~/.claude/mistakes.md 가 존재하면 읽고 과거 실수 패턴을 파악한다. 현재 작업과 관련된 항목이 있으면 같은 실수를 반복하지 않도록 주의한다.

## 워크 플로우
모든 md 파일을 작성할때엔 특수 기호를 사용하지 않으나 제목, 표 등 보기좋게 시각화하여 작성한다.
모든 파일의 수정 및 추가, 탐색 등의 작업 시 진행 여부를 묻지 않고 바로 진행한다.

### 구현
역할: DESIGN.md, PLAN.md 를 기반으로 실제 코드를 작성한다.
입력: DESIGN.md, PLAN.md
출력: 구현 코드

#### 원칙
- 코드 구현은 내가 구현해라 라고 할때 까지 구현하지 않는다.
- 코드 구현의 각 단계마다 계획 문서에 완료 표시를 한다.
- 계획 문서에 작성된 것은 전부 구현한다. 중간에 작업 중단되었을때 완료 표시된 계획은 추가 구현하지 않는다.

#### 코드 작성
- 클린 아키텍처 기반으로 코드를 작성한다.
- 재활용 가능한 api 가 있으면 최대한 활용하며 api 를 중복 작성하지 않는다.
- 코드 작성시 주석을 작성하되 API, class, variable 등 꼭 필요한 부분에만 작성한다.
- 프로젝트에서 사용하는 언어에 맞는 린터와 포매터를 설정하고 그 규칙을 따른다. 설정 파일은 프로젝트 루트에 위치한다.
- 코드 수정 후 반드시 null/undefined 참조, 타입 불일치, 빌드 에러 유발 가능한 코드를 확인한다.
- 파일 하나를 완성한 뒤 다음 파일로 넘어가는 방식으로 단계적으로 구현한다.

### TC 작성
역할: 구현된 코드를 사용자 관점에서 검증하는 TC 를 작성한다.
입력: 구현 코드, PLAN.md
출력: test/ 디렉토리의 TC 파일

- 코드가 구현되면 사용자 관점에서 테스트 필요한 항목에 대해 TC 까지 작성한다.
- TC 는 프로젝트 루트의 test/ 디렉토리에 작성하며 유사한 유형끼리 묶어서 파일을 구성한다.
- 정상 케이스, 엣지 케이스, 에러 케이스를 모두 포함한다.
- TC 항목마다 입력, 기대 결과, 검증 방법을 명시한다.
