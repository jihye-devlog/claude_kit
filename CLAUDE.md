## 워크 플로우
모든 md 파일을 작성할때엔 특수 기호를 사용하지 않고 마크다운 기본 문법만 사용한다.
모든 파일의 수정 및 추가, 탐색 등의 작업 시 진행 여부를 묻지 않고 바로 진행한다.

### 코드 리서치
역할: 코드베이스를 분석하여 구조와 동작을 파악하고 RESEARCH.md 보고서를 작성한다.
입력: 분석 대상 디렉토리 경로
출력: RESEARCH.md

- 분석이 필요한 디렉토리에 RESEARCH.md 파일이 있다면 신규 분석을 진행하지 않고 RESEARCH.md 를 확인한다.
- RESEARCH.md 가 없다면 디렉토리를 깊이 읽고 어떻게 동작하는 지 깊이 이해하고 모든 세부사항을 파악한다. 끝나면 해당 레포 또는 신규 분석한 디렉토리의 루트 경로에 RESEARCH.md 를 생성하여 상세 보고서를 작성한다.
- RESEARCH.md 에는 프로젝트 개요, 디렉토리 구조, 주요 모듈과 역할, 데이터 흐름, 의존성, 진입점, 핵심 로직, 기술 스택을 포함한다.

### 계획
역할: 사용자 요청과 RESEARCH.md 를 기반으로 구현 계획을 수립하고 PLAN.md 를 작성한다.
입력: 사용자 요청, RESEARCH.md
출력: PLAN.md

- 내가 무언가를 요청하였을 때 반드시 상세 PLAN.md 를 작성한다.
- 기능 설명, 목표, 핵심 기능, 데이터 구조, 파일 구조, UI 구성, 상세 요구사항, 접근 방식의 상세 설명, 기술 스택, 제안할 기능 등 계획 단계에 필요한 내용을 상세히 작성한다.
- 각 작업 항목은 구현 에이전트가 순서대로 실행할 수 있도록 단계별로 분리하여 작성한다.
- 불확실하거나 결정이 필요한 사항은 별도로 표시하여 사용자 확인을 요청한다.

### 설계
역할: PLAN.md 를 기반으로 실제 코드 수준의 설계를 수행하고 DESIGN.md 를 작성한다.
입력: PLAN.md, RESEARCH.md
출력: DESIGN.md

- 코드 구현 전에 반드시 DESIGN.md 를 작성한다.
- 계획 문서를 기반으로 아키텍처 개요, 상세 설계, 실제 변경될 코드 스니펫, 수정될 파일 경로, 고려될 사항이나 트레이드 오프 등 설계 단계에 필요한 내용을 상세히 작성한다.
- 인터페이스, 타입, 함수 시그니처 등 구현 에이전트가 참고할 수 있는 구체적인 코드 수준의 명세를 포함한다.
- 변경 영향 범위와 기존 코드와의 의존성을 명시한다.

### 구현
역할: DESIGN.md, PLAN.md 를 기반으로 실제 코드를 작성한다.
입력: DESIGN.md, PLAN.md
출력: 구현 코드

#### 원칙
- 코드 구현은 내가 구현해라 라고 할때 까지 구현하지 않는다.
- 코드 구현의 각 단계마다 계획 문서에 완료 표시를 한다.
- 계획 문서에 작성된 것은 전부 구현한다. 중간에 작업 중단되었을때 완료 표시된 계획은 추가 구현하지 않는다.

#### 코드 작성
- 클린 아키텍처 기반으로 코드를 작성한다.
- 재활용 가능한 api 가 있으면 최대한 활용하며 api 를 중복 작성하지 않는다.
- 코드 작성시 주석을 작성하되 API, class, variable 등 꼭 필요한 부분에만 작성한다.
- 프로젝트에서 사용하는 언어에 맞는 린터와 포매터를 설정하고 그 규칙을 따른다. 설정 파일은 프로젝트 루트에 위치한다.
- 코드 수정 후 반드시 null/undefined 참조, 타입 불일치, 빌드 에러 유발 가능한 코드를 확인한다.
- 파일 하나를 완성한 뒤 다음 파일로 넘어가는 방식으로 단계적으로 구현한다.

### TC 작성
역할: 구현된 코드를 사용자 관점에서 검증하는 TC 를 작성한다.
입력: 구현 코드, PLAN.md
출력: test/ 디렉토리의 TC 파일

- 코드가 구현되면 사용자 관점에서 테스트 필요한 항목에 대해 TC 까지 작성한다.
- TC 는 프로젝트 루트의 test/ 디렉토리에 작성하며 유사한 유형끼리 묶어서 파일을 구성한다.
- 정상 케이스, 엣지 케이스, 에러 케이스를 모두 포함한다.
- TC 항목마다 입력, 기대 결과, 검증 방법을 명시한다.

### 문서화
역할: 구현 완료 후 프로젝트 문서를 최신 상태로 유지한다.
입력: 구현 코드, PLAN.md
출력: README.md

- 코드 구현이 완료되면 README.md 에 프로젝트 구조와 설명 등을 작성하며 코드가 수정되어 설명의 수정이 필요할 경우 같이 수정한다.
- README.md 에는 프로젝트 개요, 기술 스택, 디렉토리 구조, 설치 및 실행 방법, 주요 기능, 환경 변수를 포함한다.
