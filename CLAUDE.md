## 워크 플로우
### 코드 리서치
- 레포를 새롭게 확인하거나 신규 분석이 필요할때, 디렉토리를 깊이 읽고 어떻게 동작하는 지 깊이 이해하고 모든 세부사항을 파악한다. 끝나면 해당 레포 또는 신규 분석한 디렉토리의 루트 경로에 research.md 를 생성하여 상세 보고서를 작성한다.
- 분석이 필요한 디렉토리에 research.md 파일이 있다면 신규 분석을 진행하지 않고 research.md 를 확인한다.
### 계획
- 코드를 작성하기 전에 반드시 상세 plan.md 를 작성한다. 기존에 파일이 존재하고 작성될 내용이 중복된 내용이 아니라면 하단에 추가하고 중복된 내용이라면 기존 내용을 수정한다.
- 접근 방식의 상세 설명, 실제 변경될 코드 스니펫, 수정될 파일 경로, 고려될 사항이나 트레이드 오프 등의 내용을 작성한다.
### 구현
- 코드 구현은 내가 구현해라 라고 할때 까지 구현하지 않는다.
- 작업이나 단계를 완료하면 계획 문서에서 완료로 표시한다.
- 계획 문서에 작성된 것은 전부 구현한다. 중간에 작업 중단되었을때 완료 표시된 계획은 추가 구현하지 않는다.
- 코드 수정 후 반드시 지속적으로 타입체크를 실행한다.
