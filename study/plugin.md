## skill-creator
### skill-creator 통해 스킬 생성
❯ /skill-creator:skill-creator 클로드가 실수할때마다 특정 md 파일에 기록해두거나 기억하고  다신 같은 실수를 반복하지 않게 하고싶어

⏺ 좋아. 스킬 초안을 작성하기 전에 몇 가지 확인할게.

  저장 위치
  - 프로젝트별로 다른 파일 (<레포>/.claude/memory/mistakes.md)
  - 아니면 전역으로 하나 (~/.claude/mistakes.md)

  "실수"의 범위
  - 사용자가 직접 지적/교정할 때만 기록
  - 아니면 잘못된 파일 수정, 빌드 에러 등 Claude가 스스로 판단할 때도 포함

  트리거 방식
  - 사용자가 /record-mistake 같은 커맨드를 직접 실행
  - 아니면 Claude가 교정 받는 상황을 감지하면 자동으로 기록

  기록 형식
  - 실수 내용 + 올바른 방법 + 컨텍스트(어떤 파일/기능)
  - 다른 형식 원하는 게 있어?

❯  저장 위치: 전역, 실수 범위: 클로드가 스스로 판단할 때도 포함, 트리거 방식: 커맨드 실행하거나 자동 기록 둘다. 기록 형식: 첫번째 내용으로 진행

⏺ 확인됐어. 바로 스킬 초안 작성할게.
⏺ 완료. 생성된 것들:

  lessons/lessons.md — 설치: cp lessons/lessons.md ~/.claude/commands/lessons.md

  동작:
  - 교정 발언, 빌드 실패, 자기 인식 시 ~/.claude/mistakes.md에 자동 기록
  - /lessons 실행하면 누적 실수 목록 요약 출력
  - 세션 시작 시 mistakes.md 읽어 과거 실수 파악

  CLAUDE.md — 세션 시작 규칙에 mistakes.md 로드 추가

  설치하면 바로 동작해. 실수 파일은 전역(~/.claude/mistakes.md)이라 모든 프로젝트에서 공유돼.

❯ /skill-creator:skill-creator

⏺ 안녕하세요! Skill Creator가 실행됐습니다.

  무엇을 만들고 싶으신가요? 아래 중 하나를 알려주시면 바로 시작할게요:

  1. 새 스킬 만들기 - 어떤 작업을 자동화하고 싶으신가요?
  2. 기존 스킬 개선 - 어떤 스킬을 개선하고 싶으신가요?
  3. 스킬 트리거 최적화 - 기존 스킬이 더 잘 발동되도록 description을 개선하고 싶으신가요?

### skill-creator improvement measure 요청 결과
❯ 스킬의 improvement measure 은 어케해? 예를 들어 /lessons 하기 전후?

⏺ 스킬 개선 측정은 반복(iteration) 비교 방식으로 합니다. /lessons 같은 스킬에 적용하면:

  측정 방식

  1. 테스트 케이스 실행 (with_skill vs baseline)

  - 신규 스킬: with_skill vs without_skill (스킬 없이 같은 프롬프트)
  - 기존 스킬 개선: with_skill (새 버전) vs old_skill (이전 버전 스냅샷)

  두 경우를 동시에 병렬로 실행해서 비교합니다.

  2. Assertions (정량 지표)

  각 테스트 케이스에 객관적으로 검증 가능한 체크리스트를 붙입니다.

  /lessons 예시라면:
  - 실수 내용이 mistakes.md에 기록됐는가?
  - 다음 세션에서 같은 실수를 반복하지 않는가?
  - 기록 형식이 올바른가 (rule / Why / How to apply)?

  3. Benchmark 생성

  iteration-1/benchmark.json
    - with_skill: pass_rate 70%, 23s, 8k tokens
    - without_skill: pass_rate 30%, 18s, 6k tokens

  4. Eval Viewer로 사람이 직접 비교

  generate_review.py로 브라우저에 결과를 띄워서:
  - 스킬 있을 때 / 없을 때 출력을 나란히 보고
  - 피드백 텍스트 남기기
  - iteration 2에선 이전 iteration 결과도 같이 표시

  5. 반복

  피드백 → 스킬 수정 → iteration-2 실행 → 다시 비교
