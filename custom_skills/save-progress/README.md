# save-progress

세션 진행 상황을 레포의 `.claude/memory/` 에 저장하는 스킬입니다.
저장된 파일은 git으로 관리되므로 다른 PC에서 새 세션을 열어도 동일한 작업을 이어나갈 수 있습니다.

## 설치 방법

스킬 파일을 글로벌 스킬 디렉토리에 복사합니다.

```bash
cp save-progress/save-progress.md ~/.claude/commands/save-progress.md
```

Claude Code를 재시작하면 `/save-progress` 슬래시 커맨드로 사용할 수 있습니다.

## 사용 방법

작업 중간이나 세션 종료 전에 슬래시 커맨드를 입력합니다.

```
/save-progress
```

다음 세션에서 Claude는 `CLAUDE.md` 세션 시작 규칙에 따라 `.claude/memory/MEMORY.md` 를 자동으로 읽고 이전 진행 상황을 이어받습니다.

## 동작 방식

`/save-progress` 를 실행하면 Claude가 아래 단계를 순서대로 수행합니다.

| 단계 | 내용 |
|------|------|
| 1 현재 상태 파악 | 대화 히스토리와 프로젝트 파일을 분석해 완료 작업, 현재 단계, 다음 할 일, 미결 사항을 파악 |
| 2 디렉토리 준비 | 프로젝트 루트의 `.claude/memory/` 디렉토리가 없으면 생성 |
| 3 메모리 파일 작성 | 아래 파일들을 최신 상태로 작성하거나 덮어씀 |
| 4 CLAUDE.md 확인 | 세션 시작 규칙이 없으면 CLAUDE.md 맨 앞에 추가 |
| 5 완료 보고 | 저장된 파일 목록, 현재 단계, 다음 할 일, 미결 사항을 출력 |

## 저장되는 파일

```
.claude/memory/
├── MEMORY.md               — 메모리 파일 인덱스
├── project_progress.md     — 완료 작업 / 현재 단계 / 다음 할 일 / 미결 사항
├── project_context.md      — 프로젝트 개요 / 기술 스택 / 핵심 설계 결정사항
└── feedback_workflow.md    — 사용자가 확인한 작업 방식 규칙 (최초 1회만 생성)
```

### project_progress.md

현재 어디까지 왔는지, 다음에 무엇을 해야 하는지를 기록합니다.

```
## 완료된 작업
- [x] 기능 A 구현
- [x] TC 작성

## 현재 단계
설계 단계 완료, 구현 대기 중

## 다음 할 일
1. src/api.ts 작성
2. 유닛 테스트 추가

## 미결 사항
| 번호 | 항목 | 확인 필요 |
|------|------|-----------|
| 1 | 페이지네이션 방식 | cursor vs offset |
```

### project_context.md

새 세션에서 배경으로 필요한 프로젝트 정보를 기록합니다.

```
프로젝트 개요, 확정된 기술 스택, 설계 결정사항, 모듈 구조 등
```

### feedback_workflow.md

대화 중 사용자가 확인하거나 수정한 작업 방식 규칙을 기록합니다.
이 파일은 최초 생성 시에만 작성되며, 이후에는 덮어쓰지 않습니다.
