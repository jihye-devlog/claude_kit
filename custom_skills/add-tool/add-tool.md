# add-tool

사용자가 스킬, 에이전트, 스크립트 등의 도구 파일 경로를 알려주면
현재 레포에 해당 도구를 위한 디렉토리와 README 를 생성한다.

## 입력 형식

사용자는 아래 중 하나의 형식으로 입력한다.

- `/add-tool <명령어> <파일경로>`
- `/add-tool` 실행 후 대화로 명령어와 경로 수집

명령어: 슬래시 커맨드명 또는 스크립트 실행 명령 (예: `save-progress`, `statusline.sh`)
파일경로: 도구 파일의 절대경로 또는 `~` 경로 (예: `~/.claude/commands/save-progress.md`)

## 실행 절차

### 1단계: 도구 파일 읽기

사용자가 제공한 경로의 파일을 읽어 아래 항목을 파악한다.

- 도구의 목적과 기능 요약
- 도구 유형 판별: 스킬(`.md`) / 에이전트(`.md`) / 스크립트(`.sh` 등)
- 설치 또는 설정에 필요한 대상 경로
  - 스킬: `~/.claude/commands/`
  - 에이전트: `~/.claude/agents/`
  - 스크립트: `~/.claude/` 또는 사용자 지정 경로
- 추가 설정이 필요한 경우: `settings.json` 항목 등

### 2단계: 레포 디렉토리 생성 및 파일 복사

현재 레포 루트에 명령어 이름으로 디렉토리를 생성한다.

```
<레포루트>/<명령어명>/
```

도구 파일을 해당 디렉토리에 복사한다.

### 3단계: README.md 작성

아래 구조로 README.md 를 작성한다. 도구 유형에 따라 설치 방법 섹션을 다르게 작성한다.

```
# <명령어명>

(도구의 한 줄 설명)

## 설치 방법

(유형별 설치 명령어와 추가 설정 방법)

## 사용 방법

(실행 명령 또는 슬래시 커맨드 예시)

## 동작 방식

(주요 단계 또는 처리 흐름 요약)
```

유형별 설치 방법 작성 기준:

스킬인 경우:
```bash
cp <명령어명>/<파일명> ~/.claude/commands/<파일명>
```
Claude Code 재시작 후 `/<명령어명>` 으로 사용 가능하다고 안내한다.

에이전트인 경우:
```bash
cp <명령어명>/<파일명> ~/.claude/agents/<파일명>
```
Claude Code 재시작 후 에이전트로 사용 가능하다고 안내한다.

스크립트인 경우:
```bash
cp <명령어명>/<파일명> ~/.claude/<파일명>
chmod +x ~/.claude/<파일명>
```
`settings.json` 에 추가 설정이 필요하면 해당 JSON 블록도 함께 작성한다.

### 4단계: 완료 보고

완료 후 아래를 출력한다.

- 생성된 디렉토리 경로
- 복사된 파일 경로
- README.md 경로
- 글로벌 설치 명령어 한 줄 요약
