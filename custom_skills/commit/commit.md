# commit

현재 레포의 커밋 규칙에 따라 변경사항을 커밋한다.

## 실행 절차

### 1단계: 커밋 규칙 파악

프로젝트 루트의 `.claude/COMMIT_CONVENTION.md` 를 읽는다.
파일이 없으면 기본 형식 `[CATEGORY] 설명` 을 사용한다.

### 2단계: 변경사항 파악

다음을 실행해 변경 내용을 파악한다.
- `git status` — 변경된 파일 목록
- `git diff HEAD` — 실제 변경 내용
- `git log --oneline -5` — 최근 커밋 스타일 참고

### 3단계: 커밋 메시지 작성

1단계에서 읽은 규칙에 맞게 커밋 메시지를 작성한다.
- 변경된 파일의 카테고리를 파악해 prefix 결정
- 여러 카테고리에 걸친 변경이면 복수 prefix 사용
- 설명은 변경 내용을 간결하게 요약

### 4단계: 커밋 실행

`git add` 로 변경 파일을 스테이징하고 `git commit` 으로 커밋한다.
커밋 메시지는 사용자에게 먼저 보여주고 확인 후 진행한다.
